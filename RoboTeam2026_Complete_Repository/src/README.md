# Source Code

## Structure

- `raspberry-pi/` — perception, mapping, planning, state machine, and high-level control
- `esp32/` — motor, servo, encoder, button, and low-level safety control

Every source file should contain comments explaining its purpose, inputs, outputs, and dependencies.

## Versioning

Use meaningful commit messages and create Git tags or releases for important milestones:

- `v0.1-prototype`
- `v0.2-open-challenge`
- `v0.3-obstacle-challenge`
- `v1.0-competition`
